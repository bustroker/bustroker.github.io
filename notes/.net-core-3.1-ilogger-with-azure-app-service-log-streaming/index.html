.NET Core 3.1 ILogger with Azure App Service Log Streaming

Goal

Enable real-time log monitoring in Azure App Service using ILogger<T> in .NET Core 3.1.

Install Azure Logging Package

dotnet add package Microsoft.Extensions.Logging.AzureAppServices

Configure Logging Providers

public class Program
{
    public static void Main(string[] args)
    {
        CreateHostBuilder(args).Build().Run();
    }

    public static IHostBuilder CreateHostBuilder(string[] args) =>
        Host.CreateDefaultBuilder(args)
            .ConfigureLogging(logging =>
            {
                logging.ClearProviders()
                        .AddConsole()
                        .AddAzureWebAppDiagnostics();
            })
            .ConfigureWebHostDefaults(webBuilder =>
            {
                webBuilder.UseStartup<Startup>();
            });
}

Adding AddConsole() is optional but useful for local debugging.

Configure Log Levels

Option 1: appsettings.json

{
  "Logging": {
    "LogLevel": {
      "Default": "Information",
      "Microsoft": "Warning",
      "Microsoft.Hosting.Lifetime": "Information"
    }
  },
  "AllowedHosts": "*"
}

Note: For Azure Log Streaming, the level is controlled by the portal’s Application Logging configuration, not appsettings.json.

Option 2: Code-Based Configuration

public static IHostBuilder CreateHostBuilder(string[] args) =>
    Host.CreateDefaultBuilder(args)
        .ConfigureLogging(logging =>
        {
            logging.ClearProviders()
                    .AddAzureWebAppDiagnostics()
                    .AddFilter("Default", LogLevel.Information)
                    .AddFilter("Microsoft", LogLevel.Warning)
                    .AddFilter("Microsoft.Hosting.Lifetime", LogLevel.Information);
        })
        .ConfigureWebHostDefaults(webBuilder =>
        {
            webBuilder.UseStartup<Startup>();
        });

Note: This also doesn’t affect Azure Log Streaming - portal settings take precedence.

Use the Logger

[ApiController]
[Route("[controller]")]
public class WarmupController : ControllerBase
{
    private readonly ILogger<WarmupController> _logger;

    public WarmupController(ILogger<WarmupController> logger)
    {
        _logger = logger;
    }

    [HttpGet]
    public IActionResult Get()
    {
        _logger.LogInformation("warming up.");
        return Ok(new { status = "I'm hot"});
    }
}

Configure Azure App Service

  1. Navigate to your App Service in Azure Portal
  2. Go to Settings → Configuration → General Settings
  3. Verify Stack is set to .NET Core
  4. Go to Monitoring → App Service Logs
  5. Enable Application Logging (File System)
  6. Set Level to your desired log level

View Logs

Go to Monitoring → Log Stream and trigger your endpoint. Logs appear in real-time.
