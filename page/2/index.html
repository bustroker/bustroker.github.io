Warmup is important so the users don’t get a cold start on the app.
When deploying to Azure app service, implementing the /warmup call every X minutes is not feasible as we don’t have access to actual instances directly to make the calls, so the requests will only get to the lucky instance the balancer in front of the app service decides to send the request to.
